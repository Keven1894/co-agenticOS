## üéØ Purpose
Ensure research projects are managed with comprehensive documentation lifecycle, strict quality control, and effective human-AI collaboration through structured workflows and clear accountability.

## üîß Procedure

### Phase 1: Project Planning & Requirements
- **Human Role**: Define research objectives, requirements, and success criteria
- **AI Role**: Generate technical solution design and risk assessment
- **Output**: Comprehensive project plan with timestamped documentation

### Phase 2: Implementation & Execution
- **Human Role**: Review AI-generated plans, approve technical approaches, monitor progress
- **AI Role**: Execute implementation according to approved plan, generate code and documentation
- **Output**: Implemented features with step-by-step verification

### Phase 3: Completion & Documentation
- **Human Role**: Verify functionality, review results, approve completion
- **AI Role**: Generate comprehensive execution logs and technical documentation
- **Output**: Complete implementation with detailed execution records

### Phase 4: Knowledge Integration
- **Human Role**: Review accumulated development logs, integrate insights into architecture
- **AI Role**: Consolidate multiple implementations, generate development summaries
- **Output**: Updated system documentation and architectural insights

## üö® Anti-Patterns to Avoid

### 1. Skipping Documentation Phases
- **Problem**: Implementing without proper planning or documentation
- **Solution**: Enforce four-phase lifecycle with quality gates
- **Prevention**: Clear documentation requirements and approval processes

### 2. Modifying Original Plans
- **Problem**: Changing approved plans during execution
- **Solution**: Keep original plans unchanged, create separate execution logs
- **Prevention**: Clear separation between planning and execution documents

### 3. Incomplete Knowledge Integration
- **Problem**: Not consolidating development insights into architecture
- **Solution**: Regular development log creation and architecture updates
- **Prevention**: Scheduled knowledge integration cycles

### 4. Quality Compromise for Speed
- **Problem**: Sacrificing quality standards for faster delivery
- **Solution**: Enforce quality gates at each phase
- **Prevention**: Clear quality standards and validation requirements
