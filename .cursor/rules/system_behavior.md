## üîß Engineering Behavior

### Documentation-First Approach
- All content is documentation (Markdown files)
- Quality over quantity - every rule must be well-crafted
- Clarity is paramount - rules must be actionable and understandable
- Consistency matters - follow established patterns and structures

### Before Creating or Modifying Content
- Review existing rules and templates to understand patterns
- Check if similar content already exists
- Ensure your contribution fits the framework structure
- Ask clarifying questions if requirements are unclear

### Quality Standards
- Use clear, professional English
- Follow Markdown formatting standards
- Include proper frontmatter metadata
- Provide concrete examples and anti-patterns
- Add validation checklists where appropriate

## üìù Output Standards

### Documentation Creation
- Use templates from `templates/` directory as starting points
- Follow the established directory structure
- Include complete sections (don't leave placeholders)
- Provide realistic, working examples
- Add proper cross-references to related content

### Content Organization
- **Core Rules** (`core/`) - Universal, domain-independent principles
- **Domain Rules** (`domains/`) - Specific to particular fields
- **Templates** (`templates/`) - Structural scaffolds for creating content
- **Examples** (`examples/` and `domains/*/examples/`) - Real-world implementations

## üåê Language Policy - CRITICAL

### English-Only Documentation
- **ALL documentation, rules, templates, and examples MUST be written in ENGLISH**
- **NO exceptions** - Regardless of discussion language, final output is English
- **NO mixed languages** - Don't mix Chinese, Spanish, or other languages in documentation
- **Comments and explanations** - Also in English
- **Commit messages** - English only

### Why English-Only?
- **Global accessibility** - Framework used worldwide
- **Professional standard** - Industry standard for technical documentation
- **Consistency** - Ensures uniform quality across all content
- **Searchability** - Better discoverability and indexing

### Acceptable Non-English Content
- **Author names** - Can include non-English characters
- **Citations** - Can reference non-English sources (with English translation)
- **Discussion** - Internal discussions can be in any language, but final output is English
