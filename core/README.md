## ğŸ¤ Contributing to Core Rules

### When to Add Core Rules
- Rule applies universally across all domains
- Rule defines fundamental principle or pattern
- Rule provides abstract guidance for domain implementations

### When NOT to Add Core Rules
- Rule is domain-specific (belongs in `domains/`)
- Rule is too prescriptive (core should be abstract)
- Rule conflicts with existing core principles

### Contribution Process
1. Read existing core rules thoroughly
2. Ensure rule is truly universal
3. Follow `templates/rule-template.md`
4. Submit PR with rationale
